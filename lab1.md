# Лабораторная работа №1

## Основы работы с локальным репозиторием

**Время:** 40–50 минут  
**Цель:** научиться создавать репозиторий, делать коммиты, смотреть статус, различия и историю.

* * *

## Теоретическая справка

*   **Репозиторий** – папка `.git` + рабочая директория.
*   **Рабочая директория** – файлы, которые вы видите.
*   **Индекс (staging area)** – область перед коммитом.
*   **Коммит** – снимок состояния файлов.
*   **Untracked** – новый файл, не добавленный в Git.
*   **Modified** – изменённый отслеживаемый файл.
*   **Staged** – добавленный в индекс.

* * *

## Ход работы

### Шаг 1. Создание каталога и инициализация (2 мин)

```
mkdir lab1_git_basic
cd lab1_git_basic
git init
```

**Ожидаемый вывод:** `Initialized empty Git repository in ...`

### Шаг 2. Первый файл и статус (3 мин)

```
echo "# Лабораторная работа 1" > README.md
git status
```

*Красный цвет* – файл не отслеживается.

### Шаг 3. Добавление в индекс и коммит (3 мин)

```
git add README.md
git status
git commit -m "Добавить README.md"
```

`git status` покажет зелёный цвет – файл в индексе.

### Шаг 4. Создание второго файла и частичное добавление (5 мин)

```
echo "Привет, Git!" > hello.txt
echo "Временная строка" > temp.txt
git add hello.txt
git commit -m "Добавить hello.txt"
```

### Шаг 5. Изменение файла и git diff (5 мин)

```
echo "Привет, Git! Сегодня я учусь работать с системой контроля версий." > hello.txt
git diff
git add hello.txt
git diff --staged
git commit -m "Обновить содержимое hello.txt"
```

### Шаг 6. Просмотр истории (3 мин)

```
git log --oneline --graph
```

### Шаг 7. .gitignore (5 мин)

```
echo "temp.txt" > .gitignore
echo "*.log" >> .gitignore
git add .gitignore
git commit -m "Добавить .gitignore"
```

## Самостоятельное задание (10 мин)

1.  Создайте файл `notes.txt` с тремя строками текста.
2.  Сделайте коммит.
3.  Измените вторую строку, посмотрите `git diff`.
4.  Сделайте второй коммит.
5.  Выведите историю в формате: `git log --pretty=format:"%h - %an : %s"`
