# Лабораторная работа №2

## Ветвление и слияние (fast-forward, трёхстороннее слияние, конфликты)

**Время:** 40–50 минут

* * *

## Теоретическая справка

*   **Ветка (branch)** – подвижный указатель на коммит.
*   **HEAD** – указатель на текущую ветку/коммит.
*   **Fast-forward merge** – перемещение указателя вперёд (линейная история).
*   **Трёхстороннее слияние** – создание отдельного коммита слияния.
*   **Конфликт** – два изменения в одной и той же строке одного файла.

* * *

## Ход работы

### Шаг 1. Подготовка (5 мин)

```
mkdir lab2_branching
cd lab2_branching
git init
echo "Строка 1" > file.txt
echo "Основной контент" > main.txt
git add .
git commit -m "Начальный коммит"
```

### Шаг 2. Создание ветки feature (3 мин)

```
git checkout -b feature
```

### Шаг 3. Работа в ветке feature (5 мин)

```
echo "Строка из feature" >> file.txt
echo "Данные фичи" > feature.txt
git add .
git commit -m "Добавить feature.txt и изменить file.txt"
```

*Обратите внимание:* мы **добавляем** строку в конец file.txt, а не изменяем существующую.

### Шаг 4. Работа в ветке main (5 мин)

```
git checkout main
echo "Строка из main" >> file.txt
echo "Важный файл main" > main_data.txt
git add .
git commit -m "Добавить main_data.txt и изменить file.txt"
```

*Обратите внимание:* в main мы тоже **добавляем** другую строку в конец file.txt. Изменения находятся в разных частях файла, поэтому конфликта при слиянии не будет.

### Шаг 5. Слияние feature в main (3 мин)

```
git merge feature
```

Так как обе ветки меняли `file.txt` в разных местах (добавляли строки в конец), Git выполнит **трёхстороннее слияние** автоматически, без конфликта. Будет создан коммит слияния.

### Шаг 6. Создание ветки bugfix (2 мин)

```
git checkout -b bugfix
```

### Шаг 7. В bugfix меняем ПЕРВУЮ строку (3 мин)

```
echo "Исправление bugfix: первая строка" > file.txt
git commit -am "Bugfix: изменить первую строку"
```

*Важно:* мы используем `>` (перезапись), а не `>>` (добавление). Это меняет самую первую строку файла.

### Шаг 8. В main тоже меняем ПЕРВУЮ строку (3 мин)

```
git checkout main
echo "Версия main: первая строка" > file.txt
git commit -am "Main: изменить первую строку по-другому"
```

### Шаг 9. Слияние bugfix в main — КОНФЛИКТ (5 мин)

```
git merge bugfix
```

Git сообщит о конфликте в файле `file.txt`. Вы увидите примерно такое:

```
Auto-merging file.txt
CONFLICT (content): Merge conflict in file.txt
Automatic merge failed; fix conflicts and then commit the result.
```

### Шаг 10. Разрешение конфликта (8 мин)

Откройте `file.txt` в редакторе. Вы увидите маркеры конфликта:

```
<<<<<<< HEAD
Версия main: первая строка
=======
Исправление bugfix: первая строка
>>>>>>> bugfix
```

Решите, какой текст оставить (можно оставить один из вариантов или оба). Удалите маркеры `<<<<<<<`, `=======`, `>>>>>>>`. Например, оставьте:

```
Исправление bugfix: первая строка
```

Затем завершите слияние:

```
git add file.txt
git commit -m "Слияние bugfix: разрешить конфликт"
```

### Шаг 11. Просмотр графа веток (3 мин)

```
git log --graph --oneline --all --decorate
```

Вы должны увидеть все ветки и коммиты слияния.

* * *

**🔍 Важное замечание:** Конфликт возникает только тогда, когда **одна и та же строка** (или соседние строки) изменена по-разному в двух ветках. В этом примере конфликт произошёл специально, потому что мы изменили первую строку и в main, и в bugfix.
